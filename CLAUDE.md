# CLAUDE.md — 작업 방식 (harness)

이 파일은 이 저장소에서 Claude(또는 다른 에이전트)가 작업할 때 반드시 따라야 하는 워크플로우를 정의한다.
일반적인 코드 스타일 가이드가 아니라 **작업 순서에 대한 강제 규칙**이다.

## 프로젝트 한 줄 요약
폴더 안의 16-bit unsigned little-endian RAW(및 BMP) 이미지를 순회하며, 라인 프로파일과 사각형 ROI로
Signal / Background / Noise / CNR을 측정하고, 폴더 내 모든 이미지에 동일 좌표를 배치 적용하여
결과를 팝업과 Excel로 내보내는 PyQt5 데스크톱 도구. 상세 설계는 [GUI.md](GUI.md), [PREPROCESSING.md](PREPROCESSING.md) 참고.

## 작업 순서 (반드시 이 순서를 지킬 것)

### 1단계 — 구현 전 확인
코드를 작성하기 **전에** 다음을 사용자에게 제시하고 승인을 받는다:
- 요청을 어떻게 이해했는지 요약
- 구현 방식(수정/생성할 파일, 핵심 알고리즘, UI 변경 사항)을 구체적으로 설명
- 애매한 부분이 있으면 AskUserQuestion 등으로 먼저 확인

사용자가 명시적으로 승인("진행해", "ㅇㅇ", "좋아" 등)하기 전에는 실제 코드 작성/수정을 시작하지 않는다.
단, 사용자가 이미 plan 모드 등으로 충분히 구체적인 계획에 동의한 경우는 이 단계가 충족된 것으로 간주한다.

### 2단계 — 구현
승인된 범위 내에서 구현한다.
- 코드를 작성/수정하면서, 그 변경이 GUI 구조나 동작에 영향을 주면 [GUI.md](GUI.md)를,
  전처리/측정 로직(파일 로딩, 라인 프로파일, signal/background/noise/CNR 계산, 배치 처리)에
  영향을 주면 [PREPROCESSING.md](PREPROCESSING.md)를 **같은 작업 안에서 함께 업데이트**한다.
  코드만 바뀌고 문서가 안 바뀌는 PR/커밋은 없어야 한다.
- 두 문서는 "코드를 보지 않고도 구조/로직을 파악"하는 목적이므로, 함수 시그니처·상태 머신·공식 등을
  실제 코드와 일치하게 구체적으로 적는다. 각 문서 상단의 "최종 확인" 표시(날짜/요약)도 갱신한다.

### 3단계 — QA
구현이 끝나면:
1. Agent 도구로 QA subagent(예: subagent_type "general-purpose" 또는 "Explore")를 실행하여
   실제로 코드를 실행/리뷰하며 버그를 찾는다. 가능하면 다음을 모두 점검:
   - 모듈 단독 실행/임포트가 깨지지 않는지 (특히 `measurements.py`, `image_io.py`의 순수 함수 단위 테스트)
   - GUI가 예외 없이 기동되는지
   - PREPROCESSING.md에 적힌 공식과 실제 코드가 일치하는지
   - 엣지 케이스(파일 크기 불일치, ROI가 이미지 밖, noise=0, 빈 배경 배열 등)가 죽지 않고 처리되는지
2. 발견된 버그는 코드를 고치기 **전에** [QA.md](QA.md)에 기록한다. 기록 형식:
   ```
   ## [날짜] 버그 제목
   - 증상:
   - 원인:
   - 재현 방법:
   - 수정 내용: (수정 후 채움)
   - 상태: 발견 / 수정중 / 수정완료
   ```
3. 기록 후 버그를 수정하고, QA.md의 해당 항목 "수정 내용"/"상태"를 갱신한다.
4. 수정이 GUI.md/PREPROCESSING.md에 적힌 내용과 어긋나면 문서도 같이 갱신한다.

이 3단계(확인 → 구현 → QA) 순서는 작은 수정이라도 동일하게 적용한다. 단, 오타 수정처럼
설계에 영향이 없는 trivial 변경은 1단계를 생략할 수 있다.

## 문서 지도
- [GUI.md](GUI.md) — 위젯 구성, 상태 머신(라인/사각형 드래그), signal/slot 매핑, GUI 파라미터 ↔ 계산 함수 매핑
- [PREPROCESSING.md](PREPROCESSING.md) — 파일 포맷, 라인 프로파일 알고리즘, signal/background/noise/area/CNR 공식, 배치/내보내기 흐름
- [QA.md](QA.md) — 발견된 버그와 수정 이력 (없으면 QA 단계에서 생성)
